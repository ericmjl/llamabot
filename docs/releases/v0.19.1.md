# Version v0.19.1

This release revamps ImageBot to use LiteLLM for image generation, adds a new TurboVecDocStore, and fixes the release workflow to prevent partial PyPI uploads.

## New Features

- Revamped ImageBot to use LiteLLM for image generation, supporting multiple providers including Ollama. Added a `style` parameter for controlling image aesthetics. (79166d)
- Added TurboVecDocStore backed by turbovec's IdMapIndex for fast vector similarity search. (1666c4)
- ImageBot now saves outputs to a temporary directory by default instead of the repo root. (b52f20aa)
- Added format-aware save to ImageReference using Pillow, so images are saved in the correct format. (41d3dbfb)

## Bug Fixes

- Fixed AIMessage import ordering in imagebot.py. (2741209f)
- Fixed ImageBot save tests to use valid PNG data. (f06abb8a)
- Made Ollama test fixture non-fatal to prevent CI cascade failures. (621e3e3a)
- Fixed TurboVec tests to skip when turbovec is not installed. (ae326a33)
- Made the release workflow atomic with automatic rollback to prevent partial PyPI uploads. (1417f3d0)

## Internal

- Updated Pixi to v0.69.0 and setup-pixi to v0.9.6. (5dc59a4c, 5834ae26)
- Limited CI Python versions to 3.12 and 3.13. (480f2744)

## Deprecations

- No deprecations were introduced in this version.
